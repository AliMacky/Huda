# Contributing to Huda

Thank you for your interest in contributing to Huda!  
This document outlines how to contribute effectively and respectfully.

----------

### **Create a Feature Branch**

Please avoid working directly on `main`.

```
git checkout -b feature/your-feature-name
```

Examples:

-   `feature/new-qibla-ui`
    
-   `fix/location-permission-bug`
    
----------

### **Follow the Project Style**

Please follow these guidelines:

#### **Swift Style**

-   Use Swift’s official style conventions.
    
-   Follow Apple’s Human Interface Guidelines for UI.

-   Name files and classes descriptively.
    
-   Keep code readable.

#### **UI/UX Contributions**

-   Submit screenshots or short screen recordings in your pull request.
    
-   Keep the design consistent with the app theme.
    

#### **Commit Messages**

Good commit messages follow this style:

```
feat: added new qibla accuracy indicator
fix: resolved crash on prayer calculation
docs: improved README layout
refactor: cleaned up prayer calculation logic

```

##  Reporting Bugs

If you find a bug, please include:

-   A clear description
    
-   Steps to reproduce
    
-   Screenshots or logs if helpful
    
-   Device + iOS version
    

Please open issues under:  
**Issues → New Issue → Bug Report**

----------

##  Suggesting Features

Feature requests are welcome!  
Before opening a feature issue:

-   Browse existing issues to avoid duplicates
    
-   Describe the feature and why it benefits users
    
-   Include mockups or UI examples if possible
